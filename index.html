Setup

Git

$ git --version
git version 1.9.1

Editor

$ export EDITOR=nano # emacs, gedit, kate etc

Opzetten

$ cd mijn/werkmap
$ git init lijstje
$ git init
Initialized empty Git repository in /tmp/...
$ cd lijstje/
$ git status
On branch master
Initial commit

Config

$ git config --global user.email "ber@berk.es"
$ git config --global user.name "Bèr Kessels"

Index (staging)

$ edit supermarkt.txt
$ git add supermarkt.txt
$ git status

Committing

$ git commit
$ git status

Nog een keer

$ edit supermarkt.txt # Voeg "Melk<enter>Suiker" aan toe
$ cat supermarkt.txt
  Boodschappenlijstje

  Melk
  Suiker
$ git add supermarkt.txt
$ git commit supermarkt.txt # Suiker en melk toegevoegd

Bekijken

$ git status
On branch master
nothing to commit, working directory clean

$ git log

$ git diff <eerste sha> HEAD

$ git show <sha>

Teruggaan

$ git log --oneline --decorate
$ git checkout <eerste sha>
NOTE!
$ git log --oneline --decorate
$ git checkout master # of <laatste sha>
$ git log --oneline --decorate

Terugdraaien

$ # Voeg "Eieren" toe, stage en commit
$ git revert HEAD

$ git log

Geheel verwijderen

$ git reset --hard <sha melk en eieren>

Samenvatting

  • Je werkt lokaal. Altijd.
  • git is het hoofdcommando met eigen subcommando's
  • Een commit bereid je voor en leg je vast.
  • Iedere commit heeft een uniek ID in de vorm van een SHA.
  • Er zijn aliases voor sommige commits.

Mergeconflicten

$ git branch koppen
$ git checkout koppen
$ edit supermarkt.txt # wijzig kop in "Kopen in Supermarkt"
$ git add en commit

$ git checkout master
$ edit supermarkt.txt # wijzig kop in "Kopen in Jumbo"
$ git add en commit

$ git checkout master
$ git merge koppen
# Mergeconflict in supermarkt.txt
$ git status
$ edit supermarkt.txt

<<<<<<< HEAD
Kopen in Jumbo
=======
Kopen in Supermarkt
>>>>>>> koppen

Melk
Suiker
Yoghurt

Mergeconflict oplossen

$ edit supermarkt.txt
$ git add supermarkt.txt
$ git commit

Samenvatting

  • Branches zijn een manier om parallel te werken.
  • git log --graph --all --oneline laat je metro-map zien.
  • Werk uit Branch kan al dan niet ingevoegd worden. (Merge)
  • Mergeconflicten los je op door de bestanden te bewerken.

Meer info

  • git merge --help voor uitgebreide documentatie over mergeconflicten oplossen op commandline.
  • git mergetool --help voor het inzetten van diff-tools om mergeconflicten op te lossen.
  • Pro Git Scott Chacon

Remotes

  • Repository zoals onze lokale, op een andere plek.
  • "Andere plek" kan andere map, drive, netwerk oid zijn.
  • file://, git://, ssh:// of http://.
  • Waar ik git clone example/ anderexample/ doe, kan ook git clone http://git.example.com/example.git anderexample staan.
  • Hier werken we dus met remotes op het filesystem.
  • Remotes kunnen echter overal staan.
  • Meest voorkomend: op netwerk, centraal online, op machine van je collega.
$ cd ../
$ git clone lijstje nieuwlijstje
$ cd nieuwlijstje
$ git log
$ git branch -v
$ git remote -v

origin  /tmp/guest-kk71Nw/Documents/lijstje (fetch)
origin  /tmp/guest-kk71Nw/Documents/lijstje (push)

$ cd ../lijstje
$ git remote -v

$ edit supermarkt.txt
$ git commit en add in master

# Protip:
$ echo "Kaas" >> supermarkt.txt && git commit -am "Kaas toegevoegd"
# Don't try this at home!

Pull

$ cd ../nieuwlijstje
$ git status
$ git log
$ git pull origin master
$ git log --oneline --decorate --graph

Push

$ cd ..
$ git clone --bare lijstje backuplijstje
$ ls -ahl backuplijstje
$ ls -ahl lijstje/.git

$ cd lijstje
$ git remote add origin /.../backuplijstje/
$ git remote -v

# Voeg "kaas" to, add en commit.
$ git push origin master
